WHAT IS JOB COACH AI?
=======================

Job Coach AI is a production-ready AI web application that accepts a resume and a job description as input, and in under 60 seconds generates 12 complete, professional, personalized career coaching outputs — the equivalent of hiring a career coach, resume writer, interview trainer, salary negotiator, and LinkedIn consultant all at once.

No account. No credit card. No waiting. No data stored. Just results.

ARCHITECTURE HIGHLIGHTS

| Feature                          | Implementation                                                                  | Why It Was Chosen                                               |
|----------------------------------|---------------------------------------------------------------------------------|-----------------------------------------------------------------|
| Single AI prompt for 12 outputs  | One structured Gemini prompt returns all 12 coaching items as a single JSON object | Eliminates quota waste, reduces latency, prevents partial failures |
| Direct REST API (no SDK)         | Raw fetch() calls to Gemini REST endpoints with AbortController timeouts        | SDK had version conflicts and added unnecessary overhead         |
| Model fallback chain             | gemini-flash-latest → gemini-flash-lite-latest                                  | Ensures service availability if primary model hits quota        |
| maxOutputTokens: 8192            | Raised from 4096 after JSON truncation failures in testing                      | 12 detailed outputs require large token budgets to complete     |
| Client + server rate limiting    | Server: in-memory Map per IP | Client: localStorage mirrors server logic   | Defense in depth — client reduces unnecessary server calls       |
| Security headers                 | X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy | Enterprise-grade response hardening on every API response |
| Input sanitization               | Strip HTML tags, null bytes, 7 prompt injection patterns neutralized            | Prevents malicious payload injection through resume/job text    |
| Zero data architecture           | No database, no logging of PII, process in memory only                          | PIPEDA + CCPA compliant by design, not by policy                |
| Progressive backoff              | 5/hour limit → 1h block → 3h block → 6h block on violations                    | Deters abuse while keeping service accessible to legitimate users|
| File parsing server-side         | pdf-parse + mammoth run in Next.js API route on Vercel serverless               | Keeps client lightweight, handles binary file formats securely  |

CHALLENGES FACED AND HOW THEY WERE OVERCOME
=============================================

This project was not smooth — it was a real engineering sprint with real failures. Every obstacle was documented, diagnosed, and resolved.

| No. | Challenge Encountered                              | Root Cause                                                              | How It Was Overcome                                                                         | Lesson Learned                                               |
|-----|-----------------------------------------------------|-------------------------------------------------------------------------|---------------------------------------------------------------------------------------------|--------------------------------------------------------------|
| 1   | Gemini API returned 404 on model calls             | Used model name gemini-1.5-flash which was not available on free tier   | Built a /api/test debug endpoint to discover available models. Switched to gemini-flash-latest which works | Always verify model availability before building around it  |
| 2   | Gemini API returned 429 quota errors               | gemini-2.0-flash is a paid-tier model — free API key cannot access it  | Removed gemini-2.0-flash from the model list entirely. Free tier models identified and used only | Never assume a model is free. Test with the actual API key first |
| 3   | Google AI SDK caused version conflicts             | SDK version mismatched with Next.js 14 App Router serverless environment| Removed the SDK completely. Replaced with direct fetch() REST calls — simpler and more reliable | Raw HTTP is more portable than SDK abstractions              |
| 4   | API calls timing out after 9 seconds               | Vercel's default timeout was too short for Gemini's generation latency  | Increased AbortController timeout to 55,000ms. Set Next.js maxDuration = 60 in route.js    | Generative AI is slow. Build for it, not against it         |
| 5   | sleep(15000) code causing guaranteed timeouts       | Early retry logic added artificial delays that exceeded function limits | Removed all sleep calls. Implemented immediate fallback to secondary model instead          | Retry logic must not add delays that exceed the timeout window|
| 6   | Client received "Unexpected token A" JSON error     | When Vercel timed out, it returned an HTML error page. JSON.parse() failed on HTML | Added res.ok check before calling res.json(). Now catches non-JSON responses gracefully    | Always check HTTP status before parsing response body        |
| 7   | Six parallel Gemini calls exhausting free quota     | Original design called Gemini once per output card (6-12 calls per analysis) | Redesigned to single combined prompt returning all 12 outputs as one JSON object            | One well-structured prompt beats many small prompts every time|
| 8   | "AI returned unexpected format" error in production | maxOutputTokens was set to 4096 — too small for 12 detailed JSON outputs causing truncated JSON | Raised maxOutputTokens to 8192. Added extractJSON() fallback parser to handle partial markdown wrappers | Token limits must be sized for actual output, not estimated  |
| 9   | Rotating quote text completely invisible            | CSS: filter:drop-shadow on the same element as WebkitTextFillColor:transparent cancels both | Moved drop-shadow filter to a wrapper div. Text fill color applied to inner element only   | Webkit gradient text and CSS filters cannot coexist on one element |
| 10  | npm command not found in PowerShell               | Node.js was not added to the Windows PATH environment variable          | Manually added Node.js to $env:Path in the session. Verified with node -v and npm -v       | Always verify PATH after installing Node.js on Windows      |
| 11  | Resume upload parsing failed for certain PDFs      | Some PDF files use non-standard encoding that pdf-parse cannot read     | Added try/catch around parse calls. Returns a clear error message rather than crashing      | Defensive parsing is essential for user-uploaded files       |
| 12  | White space and layout imbalance on widescreen     | Default Next.js layout had no max-width constraint, content spread too wide | Added maxWidth: 1200px container, reduced horizontal padding on all sections               | Always design for the widest common screen width, not just laptop |
