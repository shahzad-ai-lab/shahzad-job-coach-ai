## Project Summary

Job Coach AI 2026 is a free, AI-powered career coaching web application. Upload or paste your resume and a job description, click Analyze — and in ~30 seconds receive 12 comprehensive career tools generated by Google Gemini AI. No login required. No data stored on server. 100% free.

## 12 AI Output Cards

1. **Resume Score** — ATS score /100, keyword match, section-by-section breakdown
2. **Cover Letter** — 3-paragraph, confident, role-specific, ready to send
3. **Resume Rewrite** — Full rewrite with ATS-optimized keywords and action verbs
4. **Skills Gap Analysis** — Hard/soft skills matched vs. missing, top 3 actions
5. **Interview Prep** — 5 likely questions with tailored answering strategies
6. **Intro Scripts** — 1-min, 2-min, 3-min spoken intro scripts for each interview stage
7. **STAR Stories** — 3 behavioral stories in Situation-Task-Action-Result format
8. **LinkedIn Summary** — First-person About section targeting the specific role
9. **Matching Jobs** — 5 similar job titles to apply for with reasoning and keywords
10. **Thank You Email** — Post-interview follow-up with subject line and personalized body
11. **Salary Negotiation** — Market range, opening script, counteroffer script, phrases to avoid
12. **30-60-90 Day Plan** — Week-by-week onboarding roadmap + first-day checklist

### Session 2 — First Working Version (6 Cards)
**Timestamp:** March 7, 2026

**Initial Features Built:**
- Resume text area (paste or upload PDF/DOCX/TXT)
- Job posting text area
- Single Gemini API call returning 6 JSON keys
- 6 result cards: coverLetter, resumeRewrite, skillsGap, interviewPrep, starStories, linkedinSummary

**Errors Encountered & Fixed:**

#### Error 1: Wrong Model Names (404)
**Symptom:** `gemini-1.5-flash` and `gemini-pro` returned 404
**Root Cause:** These model names are not available on this API key's tier
**Fix:** Used `/v1beta/models` ListModels API endpoint to discover actual available models. Found `gemini-flash-latest` = working HTTP 200
**Code Change in route.js:**
```js
// BEFORE (broken)
const MODELS = [{ name: 'gemini-1.5-flash', timeout: 9000 }]

// AFTER (fixed)
const MODELS = [
  { name: 'gemini-flash-latest',      timeout: 55000 },
  { name: 'gemini-flash-lite-latest', timeout: 55000 },
]
```

#### Error 2: gemini-2.0-flash 429 Quota Exceeded
**Symptom:** `gemini-2.0-flash` returned 429 immediately
**Root Cause:** Free tier quota = 0 for gemini-2.0-flash (paid-only model)
**Fix:** Removed gemini-2.0-flash from MODELS array entirely, kept only gemini-flash-latest and gemini-flash-lite-latest

#### Error 3: SDK Version Mismatch
**Symptom:** `@google/generative-ai` SDK threw version/import errors
**Root Cause:** SDK version incompatibilities with Next.js 14 serverless
**Fix:** Removed SDK entirely. Switched to direct REST fetch calls:
```js
const url = `https://generativelanguage.googleapis.com/v1beta/models/${name}:generateContent?key=${apiKey}`
const res = await fetch(url, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ contents: [{ parts: [{ text: prompt }] }] })
})
```

#### Error 4: 9-Second Timeout Too Short
**Symptom:** Large prompts always timed out at exactly 9 seconds
**Root Cause:** AbortController set to 9000ms — way too short for a 6-key combined prompt
**Fix:** Increased timeout to 55000ms (55 seconds) — leaving 5s buffer for Vercel's 60s limit

#### Error 5: sleep(15000) Killing Serverless Function
**Symptom:** Function always timed out at 60 seconds
**Root Cause:** `sleep(15000)` between model retries was eating 15s per retry
**Fix:** Removed all sleep delays entirely. On model failure → immediately try next model

#### Error 6: "Unexpected token A" JSON Parse Error
**Symptom:** Client received `Unexpected token 'A'` when trying to parse response
**Root Cause:** Vercel function timed out → returned HTML error page → client tried to JSON.parse it
**Fix:** Added proper error response wrapping + client-side `res.ok` check before JSON.parse

#### Error 7: 6 Parallel Gemini Calls Exhausting Quota
**Symptom:** 5 of 6 parallel calls failed with quota errors
**Root Cause:** Original design called Gemini 6 times simultaneously — hit free tier rate limits
**Fix:** Merged all 6 sections into ONE combined prompt returning a single JSON object. 1 call instead of 6.

### Session 4 — Critical Break & Fix
**Timestamp:** March 8, 2026

**User Input:**
"it has broken again — AI returned unexpected format. Please try again."

**Root Cause Analysis:**
- 12-key prompt generates ~6,000+ tokens of output
- `maxOutputTokens: 4096` was too low — Gemini truncated the JSON mid-object
- Truncated JSON → `extractJSON()` failed to parse → "AI returned unexpected format" error

**Fix Applied:**
```js
// BEFORE (broken)
generationConfig: { temperature: 0.7, maxOutputTokens: 4096 }

// AFTER (fixed)
generationConfig: { temperature: 0.7, maxOutputTokens: 8192 }
```

## API Architecture

```
Browser (page.js)
  |
  |-- POST /api/upload  (file upload)
  |     └── pdf-parse / mammoth / text → returns { text }
  |
  |-- POST /api/analyze  (main endpoint)
        └── Gemini REST API → single prompt → 12 JSON keys
              └── gemini-flash-latest (primary)
              └── gemini-flash-lite-latest (fallback)
```

**Single Prompt Design:** All 12 outputs are generated in one Gemini call. The prompt specifies exact JSON structure with 12 keys. `extractJSON()` strips markdown fences and uses regex fallback to reliably parse the response.

**Token Budget:**
- Input: ~6,000 resume chars + ~3,000 job chars + ~2,000 prompt template = ~11,000 input tokens
- Output: `maxOutputTokens: 8192` — enough for all 12 detailed sections

## Security Architecture

### Rate Limiting
- **Server-side:** IP-based tracking via `X-Forwarded-For` header. Max 2 requests per IP per 24-hour window. Returns HTTP 429 with friendly message and reset time on exceed.
- **Client-side:** `localStorage` stores `{ count, resetTimestamp }`. UI blocks analyze button and shows countdown before even hitting the server.

### Input Security
- Strip HTML tags: `/<[^>]*>/g`
- Strip null bytes and control characters
- Block prompt injection strings: `ignore previous instructions`, `system:`, `[INST]`, etc.
- Hard payload size limit: 50KB max request body
- Text truncation hard caps: 6,000 chars resume / 3,000 chars job posting

### HTTP Security Headers
```
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

### GRC / Compliance Posture
- **Data Residency:** No user data is persisted on any server. All processing is ephemeral in-memory within the serverless function lifecycle.
- **Data Retention:** Zero. No database. No logs of user content. Data lives only in the browser session.
- **Right to Erasure:** Automatic — refresh the page and all data is gone.
- **API Key Security:** Stored only as Vercel environment variable, never in source code, never in git history.
- **Transport Security:** All traffic via HTTPS (enforced by Vercel).
- **Principle of Least Privilege:** API key has minimum required Gemini permissions only.

## Privacy Policy (In-App)

**We do NOT store your resume. We do NOT store the job description. We do NOT store the AI results.**

When you click Analyze:
1. Your resume and job text are sent to our server
2. Our server forwards them to Google Gemini AI
3. Gemini generates the 12 results
4. Results are sent back to your browser
5. Your original input is NEVER written to any database or log file
6. When you refresh or close the page, everything is gone

**Data processed by:** Google Gemini AI (subject to Google's privacy policy)
**Data stored by us:** Nothing
**Data stored on your device:** Temporary browser memory only (cleared on refresh)
